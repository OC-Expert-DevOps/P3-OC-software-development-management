## Active Decisions

- JWT HS256 access token (15min) + UUID v4 refresh token (7d, bcrypt hash in DB)
- Refresh token in HttpOnly cookie (Secure, SameSite=Strict, path /api/auth)
- Token rotation on each refresh (old token revoked)
- File storage: MinIO via AWS SDK v3 (S3-compatible)
- Soft-delete pattern: `isDeleted` flag (no physical DB delete)
- File key format: `{userId}/{uuid}-{originalname}`
- Forbidden extensions: .exe, .bat, .cmd, .sh, .ps1
- Max file size: configurable via MAX_FILE_SIZE_BYTES (default 1GB)
- File expiry: configurable via FILE_EXPIRY_DAYS_DEFAULT (default 7 days)

## Risks & Attention Points

- `npm ci` in Dockerfiles requires package-lock.json (generated on first `npm install`)
- Prisma migrations need running postgres (`docker compose run backend npx prisma migrate dev`)
- MinIO bucket auto-created on startup via `onModuleInit`
- BigInt serialization: Prisma BigInt fields may need custom serialization for JSON responses
- Refresh token lookup iterates all non-revoked tokens (acceptable for MVP, not for scale)
